# BUMDITBUL Mobile Project

## Tech Stack
- **State Management:** Riverpod
- **HTTP Client:** Dio
- **Navigation:** GoRouter
- **Architecture:** Clean Architecture

---

## Commit Message Convention
Claude does not directly commit, but only recommends commit messages.

### feat
Add new features

**Examples**
- `feat: add notification detail screen`
- `feat: add main page button component`

---

### fix
Fix bugs

**Examples**
- `fix: modify overflow that occurs when notification is clicked`

---

### refactor
Improve code structure without changing functionality

**Examples**
- `refactor: improve ElevatedButton component structure`
- `refactor: separate NotificationView widget`

---

### style
Modify UI or formatting without logic changes

**Examples**
- `style: modify card padding values`
- `style: add and remove text styles`

---

### chore
Miscellaneous tasks such as build, setup, cleanup, or deletion

**Examples**
- `chore: delete unused button components`
- `chore: app icon settings`
- `chore: remove dead code`

---

### docs
Documentation-related changes

**Examples**
- `docs: update README`
- `docs: create commit convention document`

---

### test
Add or modify test code

**Examples**
- `test: add notification test code`

---

## Additional Rules
- **Deletion / Cleanup / Setting changes** → use `chore`
- **Structural improvements / Responsibility separation** → use `refactor`
